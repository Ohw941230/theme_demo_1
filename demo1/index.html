remはなに??

HTMLで「html」または「body」にfont-sizeを指定しない場合、ブラウザは**デフォルトフォントサイズ(default font size)**で16pxを使用します。
それで1rem = 16pxになります。
Chrome、Firefox、Safari、Edgeなどのモダンブラウザはすべてデフォルト値を16pxとしています。
16pxの 62.5% = 10pxになります。

h2:自分は3rem(30pxです)

h3:自分は2rem(20pxです)

p:自分は1.6rem(16pxです)

remの強み


1.デザインを統一しやすい！
ボタンやテキストの大きさをpxでバラバラに書くと管理が大変。
remならすべて同じ基準で作れるから、スッキリ＆統一感◎！

2.レスポンシブ＆アクセシビリティに強い
ユーザーがブラウザで文字を大きくしても、
remを使っていれば全体のサイズも自然に大きくなります！

3.後からサイズをまとめて変えられる
もし html { font-size: 18px; } にしたら、
全体が自動でバランスよく大きくなる！1つの設定で全体管理ができる！

remを使う時に注意点


1.htmlの設定に全部が左右される
htmlの文字サイズを変えると、全体のサイズも変わってしまうので要注意。

2.デザイナーはpxで作ってくることが多い
例えば、24pxを2.4remに毎回変換するのが面倒なときもある。

3.1pxの線などには向いていない
すごく細かいサイズ（1pxのボーダーなど）にはremは不向き。 → この場合はpxを使った方がいい！

emはなに??

1.emはfont-sizeを指定する時は親基準、他の属性では自分自身のfont-size基準！

以下のコードはどうなるのでしょうか？

<h3>自分は 親 font-size : 30pxです。
　　<span>
　　　自分は 子供です。 font-size : 0.5em; padding : 1emです。
　　</span>
</h3>


正解は font-size : 15px padding: 15pxです。

2.emは重なる性質を持っているので使用に注意が必要！

以下のコードはどうなるのでしょうか？

<div class="pareant"> 私は font-sizle20pxです。
　　<div class="child"> 私は font-sizle 1.5emです。 paddingは 0.5emです。
　　　<div class="grand_child"> 私は font-size2emです。paddingは1emです。
　　　</div>
　　</div>
</div>

正解は　.child : font-size 30px padding : 15px ,
.grand_child : font-size : 60px, padding : 60pxです。

emの強み


1.柔軟なレイアウト
emは相対的な単位なので、親要素のfont-sizeに比例してサイズが変更されます。これにより、レスポンシブデザインにとても便利です。たとえば、親要素のfont-sizeを変更すると、子要素のサイズも自動的に変更されるため、レイアウトを簡単に変更できます。

2.フォントサイズの調整
emを使うことで、フォントサイズを相対的に設定でき、アクセシビリティが向上します。たとえば、ユーザーがブラウザでデフォルトのフォントサイズを変更すると、emを使ったテキストはその変更に比例して大きくなったり小さくなったりするため、ユーザーにとってより良い体験を提供できます。

3.レスポンシブデザイン
emを使うと、すべての要素のサイズを比例的に調整できるため、異なる画面サイズに合わせて柔軟なデザインを作成できます。例えば、emを使ってパディングやマージン、フォントサイズを設定すると、画面サイズが変わるたびに自然にサイズが調整されます。

emを使う時に注意点


1.ネストされたemの計算
emは継承されて計算されるため、ネストが深くなるほど、予期しない大きな値が計算されることがあります。例えば、子要素のフォントサイズが親要素のfont-sizeに比例して大きくなるのですが、その中にさらに子要素がある場合、その子要素は親要素のサイズに比例してさらに大きくなるため、計算が複雑になることがあります。

2.不規則なサイズの変化
emを多用すると、デザインが不規則に変化することがあります。例えば、padding、margin、font-sizeなどをすべてemで設定していると、ブラウザのデフォルトのfont-sizeが異なる場合、予期した通りにデザインが変わらないことがあります。特に、emを多用してネストすると、レイアウトのサイズが不規則に変わる可能性があるため、注意が必要です。

3.他の単位との混在による混乱
emとpx、remなどを混ぜて使うと、予期せぬ動作をすることがあります。pxは絶対的なサイズ単位で、remはルート要素（html）のfont-sizeを基準にする相対単位です。このように、emと異なる単位を組み合わせて使うと、相対基準が異なるため、予期しない結果が生じることがあります。

ユーザーのデフォルトフォントサイズ
emはユーザーのデフォルトフォントサイズを基準にするため、デフォルトのフォントサイズが大きすぎると、サイト全体が予想以上に大きくなったり小さくなったりすることがあります。たとえば、ユーザーがブラウザでフォントサイズを変更した場合、emを使ったすべての要素がその変更に基づいてサイズが調整されます。

まとめ

・remは、htmlまたはbodyのfont-sizeを基準にして、全体的なサイズを統一できます。
レスポンシブデザインやアクセシビリティに優れ、後から一括で調整できます。

・emは、親要素のfont-sizeを基準にして柔軟なレイアウトが可能ですが、自己のフォントサイズも影響するため、paddingやmarginに予期しない変化が生じることがあります。
特にネストが深くなると、サイズが重なって計算が複雑になりやすいので注意が必要です。
